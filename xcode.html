Xcode 12 新特性

# 作为 iOS，macOS，tvOS 和 watchOS 开发的中心地带，经过多年的迭代，Xcode 如今已经有了许多显著的提升。尽管 iPad 版 Xcode 今年又无望了，不过 Xcode 12 的确收获了一些激动人心的更新。

# 在接下来的部分，我们一起来看看这个 IDE 除了重制的图标，还给我们带来了哪些新功能。

# 新的 SwiftUI 应用生命周期

# 当你创建新的 Xcode 项目时，会看到新的开始模板，可以让你从 SwiftUI App 和 UIKit App Delegate 中做出选择。

# 当你选择了 SwiftUI App，你创建的工程将不会有 App Delegate 和 Scene Delegate，取而代之的是以 @main 属性标记的应用主入口（这个特性由 Swift 5.3 引入）

# 新的文档 Tab 和外观变化

# 开发者们对文档 tab 期待已久。新特性可以让我们通过双击快速地打开不同的文件到新的 tab，并且可以重新排列它们。假如你把一个文件夹拖进 tab bar，该文件夹下所有的文件都会被打开。

# Tab bar 是可滚动的，当打开的 tab 很多时，可以通过滚动来查看不同的 tab。

# 项目导航器的图标尺寸和文本字号现在可以通过 Xcode 设置中的 “Sidebar Font & Icon Size” 的选项来调节。

# Assets.xcassets 文件夹引入了 accentColor，可以让我们为应用设置全局的主题色，而且现在也直接支持 SVG 矢量图了。

# 文章开头提到，新版本的工具栏和侧边栏的图标也全都重新设计了，以加强在 macOS Big Sur 上的用户体验。其中左边栏的触发按钮被移到了左边。

# 指标丰富的 Organizer

# Organizer 现在不仅可以上传和查看产品归档，还可以查看应用的各种分析细节 —— 从电量使用到启动时间，内存使用，再到 scroll-hitch 指标 —— 涵盖你需要用于诊断问题和优化应用的一切手段。

# 来自 WWDC20 视频

# Devices and Simulators 弹出的 “Add Device” 窗口现在也可以调节大小了。

# 健壮的自动补全

# 自动补全一直是 Xcode 的软肋，有的时候，自动补全加载的时间会很长。这一致命弱点已经被 Xcode 12 以闪电般的自带默认值的补全效果彻底粉碎了。

# 新的自动补全效果奇怪的地方是它不会提供给你默认的 () 选项。因此，你不得不手动清除。希望这个问题在将来的版本会得到修复。

# SwiftUI 预览和代码自动完成

# 模拟器的增强
